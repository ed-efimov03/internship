# Основы ООП (OOP/base-oop)

## Задачи и их решения

Этот репозиторий содержит несколько примеров реализации абстрактных классов и принципов ООП в Python, иллюстрирующих использование абстракций, декораторов и наследования. Рассмотрим каждый пример более подробно.

### **Пример 1: Абстрактные классы для космических кораблей**
В этом примере создаются абстрактные классы для космических кораблей (например, FederationStarship и KlingonWarship), которые реализуют обязательные методы для управления движением, оружием и системой самоуничтожения.

**Основные принципы:**
- **Абстракция**: использование абстрактных классов для создания базовых интерфейсов.
- **Наследование**: классы, такие как FederationStarship и KlingonWarship, наследуют от абстрактного класса Starship и реализуют методы для конкретных типов кораблей.

**Пример кода:**
```python
class Starship(ABC):
    @abstractmethod
    def warp_speed(self):
        pass
    @abstractmethod
    def fire_weapon(self):
        pass
    @abstractmethod
    def self_destruct(self):
        pass
```

### **Пример 2: Абстрактные классы для ингредиентов**
В этом примере создаются абстрактные классы для работы с ингредиентами (овощи и фрукты). Классы Vegetable и Fruit реализуют интерфейс для получения данных об ингредиентах.

**Основные принципы:**
- **Абстракция**: базовый класс Ingredient служит интерфейсом для других типов.
- **Наследование**: классы Vegetable и Fruit реализуют методы для конкретных типов ингредиентов.

**Пример кода:**
```python
class Ingredient(ABC):
    @abstractmethod
    def get_name(self):
        pass
    @abstractmethod
    def get_quantity(self):
        pass
```

### **Пример 3: Декоратор для проверки типа объекта**
Этот пример показывает, как можно использовать декораторы для проверки типа объекта перед выполнением метода. В данном случае декоратор validate_soldier проверяет, является ли объект экземпляром класса Soldier.

**Основные принципы:**
- **Декораторы**: использование декораторов для проверки условий выполнения метода.
- **Абстракция и наследование**: базовый класс Soldier и его наследники (Infantry и Cavalry) реализуют общие методы для солдат.

**Пример кода:**
```python
def validate_soldier(func):
    def wrapper(self):
        if not isinstance(self, Soldier):
            raise TypeError("Объект не является экземпляром Soldier")
        return func(self)
    return wrapper
```

### **Пример 4: Абстрактные классы для динозавров**
Здесь создаются абстрактные классы для динозавров, с возможностью фильтрации по типу рациона (плотоядные и травоядные). Каждый динозавр имеет методы для получения информации о своем типе и рационе.

**Основные принципы:**
- **Абстракция**: базовый класс Dinosaur задает обязательные методы для получения информации о динозавре.
- **Наследование**: классы Carnivore и Herbivore реализуют методы для разных типов динозавров.

**Пример кода:**
```python
class Dinosaur(ABC):
    @abstractmethod
    def get_personal_name(self):
        pass
    @abstractmethod
    def get_breed(self):
        pass
    @abstractmethod
    def get_height(self):
        pass
    @abstractmethod
    def get_weight(self):
        pass
    @abstractmethod
    def get_diet(self):
        pass
```

### **Пример 5: Музыкальные инструменты и оркестр**
В этом примере создается иерархия классов для музыкальных инструментов. Инструменты могут быть струнными или ударными, и каждый инструмент может воспроизводить звук.

**Основные принципы:**
- **Абстракция**: базовый класс Instrument задает обязательные методы для всех инструментов.
- **Наследование**: классы StringedInstrument и PercussionInstrument реализуют конкретные типы инструментов.
- **Полиморфизм**: методы play реализуются для каждого типа инструмента.

**Пример кода:**
```python
class Instrument(ABC):
    @abstractmethod
    def get_name(self):
        pass
    @abstractmethod
    def get_type(self):
        pass
    @abstractmethod
    def get_sound(self):
        pass
    @abstractmethod
    def play(self):
        pass
```

### **Задача 6: Фильмы и их жанры**

**Основные принципы:**
- Наследование: создание разных типов фильмов (ужасы, боевик, мелодрама и т. д.) на основе базового класса `Movie`.
- Полиморфизм: переопределение метода `play` в каждом классе-наследнике для отображения специфичного для жанра сообщения.

**Описание кода:**
- Класс `Movie` является базовым для всех типов фильмов. Он инициализирует название фильма и его режиссера, а также метод `play`, который выводит сообщение о просмотре фильма.
- Классы-наследники (например, `Horror`, `Action`, `Romance`) переопределяют метод `play` для вывода специфических сообщений.
- `FilmCatalogue` представляет собой каталог фильмов, с возможностью добавления фильмов, поиска и воспроизведения фильмов по жанрам.

**Пример кода:**
```python
my_catalogue = FilmCatalogue()
my_catalogue.add_movie(Drama("Крестный отец", "Френсис Ф. Коппола"))
my_catalogue.play_all_movies()
```

### **Задача 10: Проверка возможности майнинга криптовалюты**

**Основные принципы:**
- Декоратор `minable_required` проверяет, можно ли майнить криптовалюту, и если нет, выводит сообщение.
- Реализация разных криптовалют с возможностью майнинга (Nano, Iota, Stellar).

**Описание кода:**
- Классы криптовалют (`Nano`, `Iota`, `Stellar`) используют декоратор `minable_required`, чтобы проверить, можно ли майнить данную криптовалюту.
- Для майнящих криптовалют (например, Nano и Iota) метод `mining_reward` возвращает награду.

**Пример кода:**
```python
cryptocurrencies = [Nano(block_lattice=True), Iota(tangle=True)]
for crypto in cryptocurrencies:
    print_info(crypto)
    if crypto.minable:
        reward = crypto.mining_reward()
        print(f"Награда за майнинг: {reward} {crypto.symbol}")
```
