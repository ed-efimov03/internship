### **Пример 1: Абстрактные классы для космических кораблей**
В этом примере создаются абстрактные классы для космических кораблей (например, FederationStarship и KlingonWarship), которые реализуют обязательные методы для управления движением, оружием и системой самоуничтожения.

**Основные принципы:**
- **Абстракция**: использование абстрактных классов для создания базовых интерфейсов.
- **Наследование**: классы, такие как FederationStarship и KlingonWarship, наследуют от абстрактного класса Starship и реализуют методы для конкретных типов кораблей.

**Пример кода:**
```python
class Starship(ABC):
    @abstractmethod
    def warp_speed(self):
        pass
    @abstractmethod
    def fire_weapon(self):
        pass
    @abstractmethod
    def self_destruct(self):
        pass
```

### **Пример 3: Декоратор для проверки типа объекта**
Этот пример показывает, как можно использовать декораторы для проверки типа объекта перед выполнением метода. В данном случае декоратор validate_soldier проверяет, является ли объект экземпляром класса Soldier.

**Основные принципы:**
- **Декораторы**: использование декораторов для проверки условий выполнения метода.
- **Абстракция и наследование**: базовый класс Soldier и его наследники (Infantry и Cavalry) реализуют общие методы для солдат.

**Пример кода:**
```python
def validate_soldier(func):
    def wrapper(self):
        if not isinstance(self, Soldier):
            raise TypeError("Объект не является экземпляром Soldier")
        return func(self)
    return wrapper
```

### **Задача 1: Музыкальный альбом

#### Задача:
Создать класс `MusicAlbum`, который представляет музыкальный альбом, включая название альбома, исполнителя, год выпуска, жанр и список треков. Также добавить метод для воспроизведения случайного трека из альбома.

#### Решение:
Класс `MusicAlbum` принимает атрибуты и имеет метод `play_random_track`, который выбирает и выводит случайный трек из списка. Для выбора случайного трека используется функция `randint` из модуля `random`.

#### Ключевые принципы:
- Определение класса
- Конструктор (`__init__`) с валидацией данных
- Метод для случайной выборки

#### Пример кода:
```python
from random import randint

class MusicAlbum:
    def __init__(self, title, artist, release_year, genre, tracklist):
        if not(isinstance(title, str) and isinstance(artist, str) and isinstance(release_year, int) 
            and isinstance(genre, str) and isinstance(tracklist, list)):
            raise ValueError("Невалидные значения")
        self.title = title
        self.artist = artist
        self.release_year = release_year
        self.genre = genre
        self.tracklist = tracklist

    def play_random_track(self):
        random_track_number = randint(1, len(self.tracklist)) 
        random_track = self.tracklist[random_track_number - 1] 
        print(f"Воспроизводится трек {random_track_number}: {random_track}")
```

Создать класс `Publisher`, который хранит информацию об издательстве и симулирует процесс публикации. Также создать подклассы `BookPublisher` и `NewspaperPublisher`, которые добавляют функциональность для работы с книгами и газетами соответственно.

#### Решение:
Базовый класс `Publisher` хранит название и местоположение. Подклассы `BookPublisher` и `NewspaperPublisher` расширяют этот класс, добавляя атрибуты и методы, специфичные для книг и газет.

#### Ключевые принципы:

### **Задача 5: Банковский счёт

#### Задача:
Создать класс `BankAccount`, который управляет балансом банковского счёта, процентной ставкой и позволяет вносить деньги, снимать деньги и добавлять проценты.

#### Решение:
Класс `BankAccount` имеет методы для внесения и снятия денег, добавления процентов и вывода истории транзакций.

#### Ключевые принципы:
- Инкапсуляция (использование приватных атрибутов)
- Отслеживание истории транзакций
- Расчёт процентов

#### Пример кода:
```python
class BankAccount:
    def __init__(self, balance, interest_rate):
        if not((isinstance(balance, int) or isinstance(balance, float)) and 
               (isinstance(interest_rate, int) or isinstance(interest_rate, float))):
            raise ValueError("Невалидные значения")
        self.__balance = balance
        self.__interest_rate = interest_rate
        self.__transactions = []

    def deposit(self, amount):
        self.__balance += amount
        self.__transactions.append(f"Внесено {amount}")

    def withdraw(self, amount):
        if amount <= self.__balance:
            self.__balance -= amount
            self.__transactions.append(f"Снято {amount}")
        else:
            print("Недостаточно средств")

    def add_interest(self):
        interest = self.__balance * self.__interest_rate
        self.__balance += interest
        self.__transactions.append(f"Начислены проценты: {interest}")

    def history(self):
        print("\n".join(self.__transactions))
```

   **Задача**: Реализация библиотеки с возможностью добавления книг и вывода списка доступных книг.
   
   **Решение**:
   - В классе `Library` используется словарь, в котором ключами являются объекты класса `Book`, а значениями — количество экземпляров этой книги в библиотеке.
   - Метод `add_book` добавляет книгу в библиотеку или увеличивает количество существующей.
   - Метод `show_books` выводит список всех книг в библиотеке и их количество.

   **Принцип**: Использование словаря позволяет эффективно хранить книги и их количество, что удобно для учета.

   **Пример**:
   ```python
   library = Library()
   library.add_book(book1)
   library.add_book(book2)
   library.show_books()
   ```

### 3. **Класс Reader:**
   **Задача**: Реализация читателя, который может взять книгу из библиотеки и вернуть ее обратно.
   
   **Решение**:
   - Класс `Reader` содержит информацию о читателе (имя, фамилия, возраст).
   - Методы `take_book` и `return_book` позволяют читателю взять книгу из библиотеки или вернуть ее.

   **Принцип**: Взаимодействие с библиотекой через методы, которые проверяют доступность книги в библиотеке перед ее выдачей и возвращением.

   **Пример**:
   ```python
   reader = Reader("Ed", "Efimov", 21)
   reader.take_book(book1, library)
   reader.return_book(book1, library)
   ```

## Пример использования

```python
# Создание библиотеки и книг
library = Library()

book1 = Book("1984", "George Orwell", 1949)
book2 = Book("To Kill a Mockingbird", "Harper Lee", 1960)
book3 = Book("The Great Gatsby", "F. Scott Fitzgerald", 1925)

# Добавление книг в библиотеку
library.add_book(book1)
library.add_book(book2)
library.add_book(book3)

# Отображение книг в библиотеке
library.show_books()

# Создание читателя
reader = Reader("Ed", "Efimov", 21)

# Читатель берет книгу
reader.take_book(book2, library)

# Отображение книг после взятия
library.show_books()

# Читатель возвращает книгу
reader.return_book(book2, library)

# Отображение книг после возврата
library.show_books()
```

### Основные принципы:
- **Классы и инкапсуляция**: Каждый объект (книга, библиотека, читатель) инкапсулирует свою собственную логику и данные.
- **Переопределение методов**: Методы `__eq__` и `__hash__` используются для сравнения и хранения объектов в коллекциях.
- **Работа с коллекциями**: Использование словаря для учета количества книг в библиотеке.
